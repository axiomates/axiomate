# Harness 模式 Hermes Axiomate 差异说明
1 Tool Call Repair Intermediate 更强 ++ Hermes: fuzzy name match (difflib 0.7) + JSON validation + truncation detection + 3-retry。Axiomate: schemaGuidedRepair.ts（~1,300 行，Hungarian 全局分配） + tolerantJsonParser.ts (21 种 repair kind + 评分契约测试) + rectangularAssignment.ts（Kuhn-Munkres + property test）+ 类型强转链 + toolCallFailureCounter.ts（连续失败升级 hint）+ structured 诊断透传 LLM
2 Error Classification Advanced Advanced Hermes: 14 FailoverReason + recovery hints。Axiomate: errorClassifier.ts (642 行) 18 reasons + 3 hints (retryable/shouldCompress/shouldFallback) + 中文识别
3 Retry + Backoff Advanced Advanced 两者都有 exponential backoff + jitter。Axiomate withRetry.ts (395 行) 额外有 persistent retry mode (unattended) 和 consecutive 529 tracking
4 Context Compression Advanced Partial GAP Hermes context_compressor.py (1,216 行): LLM-assisted summarization + tool-call JSON integrity。Axiomate compact.ts (1,468 行): 4 策略 (full/micro/reactive/snip)，缺 iterative summary template
5 Prompt Caching Intermediate Advanced Axiomate promptCacheBreakDetection.ts (634 行): hash-based cache break detection + per-tool schema diff。Hermes: 4 cache_control breakpoints, ephemeral 5m TTL
6 Credential Pooling Advanced 缺失 GAP Hermes credential_pool.py (1,326 行): 多 credential per provider + 4 rotation strategy + OAuth refresh。Axiomate: 单 key per model
9 Smart Model Routing ⊘ DELETED N/A CANCELLED Hermes 在 commit 424e9f36（2026-04-19）整体删除 smart_model_routing.py（opt-in 默认关 + 用户少 + 维护贵）。axiomate 不应 port —— hermes 用产品事实证明 ROI 不够。替代：/fast slash command 让用户主动选 fastModel
10 Tool Result Size Mgmt Intermediate 更强 Axiomate toolResultStorage.ts (983 行): per-tool 50K + per-message 200K + disk overflow + token budget enforcement。Hermes: per-tool max_result_size_chars
11 Parallel Tool Execution Advanced Advanced Hermes: path-overlap detection + 8 workers + checkpoint。Axiomate StreamingToolExecutor.ts: concurrent-safe 分类 + sibling abort 在 Bash 错误时
13 Iteration Budget Intermediate 更强 Axiomate query.ts: diminishing returns + continuation heuristics + multi-factor exhaustion。Hermes: simple counter (max 90 parent / 50 child)
14 Stale Connection Detection Intermediate Intermediate 两者功能相当。Hermes: last_chunk_time + dynamic stale timeout (300-600s)。Axiomate stallDetection.ts (71 行) + ECONNRESET/EPIPE 检测
15 Provider Abstraction Advanced Intermediate Hermes: 3 API modes (chat_completions/anthropic_messages/codex_responses) + 6-level auxiliary fallback chain。Axiomate provider.ts: neutral StreamRequest/Response，anthropicProvider.ts (675 行) + openaiProvider.ts (425 行)，无 fallback 链
16 Subagent / Delegation Advanced Partial Hermes delegate_tool.py: credential inheritance + toolset restriction + heartbeat (30s) + depth limit=2。Axiomate AgentTool/runAgent.ts + forkedAgent.ts: 状态隔离但缺 depth limit / heartbeat / credential inheritance
17 Cross-Session Memory Advanced Partial Hermes memory_manager.py (373 行): pluggable providers + lifecycle hooks。Axiomate sessionMemory.ts (450 行): file-based auto-memory，无 provider 架构
18 Auxiliary LLM Client Advanced 部分实现 Hermes auxiliary_client.py (2,771 行): 6 级 fallback + per-task override。Axiomate: 三级模型 (currentModel / midModel / fastModel) + 自动 fallback 链 + 内部任务路由 cheap tier — 重叠 >70%。差距：3 级 vs 6 级、无 per-task 配置表、无 credit exhaustion tracking、无 vision provider 单独 resolution
19 Skill & Memory Auto-Distillation Advanced 部分实现 Hermes 3 层架构：SKILLS_GUIDANCE 系统提示 + skill_manager_tool.py (817 行) 6 action 暴露给 LLM 自主写 + 每 N 轮 (默认 10) 后台 fork-agent 跑 review prompt 自动决策存/更/不存。Axiomate 有 /skillify bundled skill（用户主动）+ skill loader + chokidar watcher + forkedAgent 基础设施全在；缺 LLM 自主版 skill_manage tool + 周期 fork-review 触发器
20 Lifecycle Hooks / Plugin System Advanced ✅ 完整 Axiomate 实现：agent/src/types/hooks.ts 11+ event types（PreToolUse / PostToolUse / SessionStart / Setup / UserPromptSubmit / PermissionRequest / FileChanged / CwdChanged / Notification 等）+ agent/src/utils/hooks/ 17 文件（agent subprocess / HTTP POST + ssrfGuard / LLM-prompt 三轨执行）+ /hooks slash command。结构上比 hermes 更丰富（hermes 仅 shell subprocess）
21 Auto-Recall / Pre-Turn Memory Prefetch Intermediate 缺失 GAP Hermes plugins/memory/hindsight/__init__.py 配 auto_recall: true 时每轮前 vector + 知识图检索 → 注入 <memory-context> 块给 system；可配 budget low/mid/high。Axiomate sessionMemory.ts 是被动 file-based，不做 pre-turn 检索
22 Session Title Auto-Generation Basic WONTFIX Axiomate REPL 已生成 title 但只存内存（终端窗口标题用），SDK 路径有 saveAiGeneratedTitle 但要显式请求。判定持久化不值得：title 信息密度低（仅反映首条提问） / Agent 不消费 / /resume firstPrompt 截断已够 / 持久化收益不抵心智+维护成本
23 Session Search Tool Advanced Advanced 独立 LLM-facing 工具，与 WebSearchTool/Bash 同级。Hermes tools/session_search_tool.py (590 行)：FTS5 BM25 + 父链 walk + 默认 per-session aux LLM 5-段式 recap。Axiomate tools/SessionSearchTool/ (1244 行, 9 文件): JSONL 流式扫描 + 4-stage 算法 (mtime → metadata tail → 全文 → rank/snippet) + snippet 默认零 LLM（hermes 默认每次摘要）+ include_summary 显式 opt-in + 默认包含当前 session（hermes 排除）+ CJK substring 友好 + shouldDefer:true 渐进发现
24 PTC (Programmatic Tool Calling) Advanced 缺失 CANDIDATE Hermes tools/code_execution_tool.py：LLM 一次写 Python script，通过 UDS RPC 在子进程内调多个 tool（read_file / web_search / terminal 等），中间结果不进 LLM context。5 tool 串联可省 50%+ token + 60%+ 延迟。Hermes Windows disable（UDS POSIX-only）。axiomate 跨平台拿下 = frontier 差异化。未来候选，不立刻启动（替换 IPC 为 namedpipe / TCP localhost；sandbox 设计）
25 OS-level Sandbox (Bash 隔离) 无 独有 axiomate sandbox-axiomate（bwrap+seccomp / sandbox-exec）—— 真 OS-level 安全沙箱，隔离 Bash fs/网络访问。Hermes 没有等价物（hermes 的"sandbox" 仅指 PTC 进程隔离 + 资源上限，不是安全隔离）。Windows 两边都没真沙箱（行业现实，OS 缺 unprivileged sandbox primitives）
