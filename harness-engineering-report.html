# Harness 模式 Hermes Axiomate 差异说明
6 Credential Pooling Advanced 缺失 GAP Hermes credential_pool.py (1,326 行): 多 credential per provider + 4 rotation strategy + OAuth refresh。Axiomate: 单 key per model
14 Stale Connection Detection Intermediate Intermediate 角色不同：hermes 主动 killer + 智能阈值；axiomate 被动 observer + 事后 retry。

Hermes 独有 4 项（run_agent.py:6222+）：
1. 3 级阈值 fallback 配置：providers.<id>.models.<model>.stale_timeout_seconds → providers.<id>.stale_timeout_seconds → HERMES_STREAM_STALE_TIMEOUT env
2. 本地 endpoint 自动 disable（Ollama / mlx / llama-cpp 检测到 → 阈值设无穷大；本地 prefill 长 context 可能 5+ 分钟）
3. 大 context 动态扩展：>50k token → max(180s, 240s)；>100k token → max(180s, 300s)
4. 超阈值主动 kill httpx + retry hand-off（不等 OS socket timeout）+ 30s gateway heartbeat（防上游 gateway inactivity 误杀）

Axiomate（middleware/stallDetection.ts 71 LOC + errorClassifier.ts）：
- 单层 chunk-gap 监测，固定 30s threshold（不可配）
- 纯 observational：仅 onStall callback 报 log，不主动 kill 连接
- 靠 Node TCP keepalive / OS socket 自然 timeout → ECONNRESET → withRetry.ts 重试（事后处理）

axiomate 实际 OK 的原因：CLI 无 gateway inactivity 问题（hermes heartbeat 是给 Telegram/Slack/Discord 网关用的）；transport error 由 errorClassifier 接住后 withRetry 重试，效果上等价。

边缘 case 慢于 hermes：① 本地 endpoint（Ollama 长 prefill）—— hermes 自动 disable，axiomate 30s 触发误判但不 kill 连接，日志噪音；② 100k+ context + Opus 慢思考 —— hermes 动态阈值放宽，axiomate 同样误报但靠 OS socket timeout 兜底
15 Provider Abstraction Advanced Intermediate Hermes: 3 API modes (chat_completions/anthropic_messages/codex_responses) + 6-level auxiliary fallback chain。Axiomate provider.ts: neutral StreamRequest/Response，anthropicProvider.ts (675 行) + openaiProvider.ts (425 行)，无 fallback 链
16 Subagent / Delegation Advanced Partial Hermes delegate_tool.py: credential inheritance + toolset restriction + heartbeat (30s) + depth limit=2。Axiomate AgentTool/runAgent.ts + forkedAgent.ts: 状态隔离但缺 depth limit / heartbeat / credential inheritance
17 Cross-Session Memory Advanced Partial Hermes memory_manager.py (373 行): pluggable providers + lifecycle hooks。Axiomate sessionMemory.ts (450 行): file-based auto-memory，无 provider 架构
18 Auxiliary LLM Client Advanced 部分实现 Hermes auxiliary_client.py (3,144 行): 6 级 fallback + per-task override。Axiomate: 三级模型 (currentModel / midModel / fastModel) + 自动 fallback 链 + 内部任务路由 cheap tier + per-model extraParams / thinkingParams + openaiRequestAdapter 跨格式转换 + errorClassifier 反应式 max_tokens 回退 — 重叠 >85%。

真 gap（4 项）：
1. 3 级 vs 6 级 fallback chain（hermes 多 primary aux / secondary aux / local 三层）
2. per-task 配置表（hermes 可按 task 类型单独配 timeout / extra_body / 模型；axiomate 是 per-MODEL extraParams，不同抽象层）
3. 无 credit exhaustion → fallback 联动（axiomate errorClassifier 能分辨 billing/auth，但未自动切下一 provider）
4. 无 vision provider 单独 resolution（hermes resolve_vision_provider_client() + _PROVIDER_VISION_MODELS 映射 xiaomi→mimo-v2.5 / zai→glm-5v-turbo；axiomate sideQuery provider-agnostic）

已澄清的"看似 gap"实际等价：
- per-model extra params：config.ts:280 extraParams + thinkingParams，openaiProvider.ts:88 Object.assign(body, extraParams) 透传。axiomate per-MODEL 抽象比 hermes per-TASK 更贴现实（特殊参数是模型属性）
- 跨 provider 图像格式：openaiRequestAdapter.ts:90-98 内部 image block 自动转 OpenAI image_url（实测 OpenAI 模式发图识别正常）
- max_tokens 参数名：errorClassifier.ts:133-137 反应式回退；仅"直连 api.openai.com + o-系列 reasoning 模型"才需要 max_completion_tokens，axiomate 典型用户（SF / OR / Qwen / 本地）不触发
- 模型特定参数约束（Kimi temperature / Opus 4.7 sampling）：用户在 ~/.axiomate.json per-model extraParams 显式覆盖（declarative，hermes 是 auto-detect）

不适用 axiomate 的 4 项：Nous Portal 401 refresh（不接 Nous）/ Codex Responses adapter（仅支持 Anthropic + OpenAI 协议）/ Python asyncio loop cache（Node 无等价问题）/ Nested AsyncOpenAI cleanup（Python httpx 特有）

核心架构差异：hermes per-TASK 抽象（compression / vision / search 各成一等公民可独立配）vs axiomate per-MODEL 抽象（特殊参数随模型走）。前者对"vision 慢便宜可等"这类任务级偏好更优；后者对"Kimi 不准发 temperature"这类模型级约束更优。两套抽象正交，不是单纯优劣关系
19 Skill & Memory Auto-Distillation Advanced 部分实现 Hermes 3 层架构：SKILLS_GUIDANCE 系统提示 + skill_manager_tool.py (817 行) 6 action 暴露给 LLM 自主写 + 每 N 轮 (默认 10) 后台 fork-agent 跑 review prompt 自动决策存/更/不存。Axiomate 有 /skillify bundled skill（用户主动）+ skill loader + chokidar watcher + forkedAgent 基础设施全在；缺 LLM 自主版 skill_manage tool + 周期 fork-review 触发器
