Checkpoints — 自动文件快照与回滚

Axiomate 在你和它对话的每一回合，都会偷偷给工作目录拍一张快照，存进 一个独立的 git 影子仓库里。如果你后悔了某次改动，/rewind 把工作目录回到任意一个历史回合，连带把对话也一起回到那个时刻。整个 系统对你近乎透明 — 平时几乎感觉不到，关键时刻它替你兜底。

1它怎么工作

Checkpoints 是一个独立的 git 仓库，住在 ~/.axiomate/checkpoints/store/，跟你项目本身的 git 仓库完全分开。Axiomate 每次准备改文件之前，会在这个 影子仓库里写一个 commit 来记录当前状态。

关键设计

影子仓库

不污染你项目自己的 .git/。你的 git log、git status 都看不到这些快照。

每个项目一条 ref

不同的工作目录写到不同的 refs/axiomate/<hash>，互不干扰。

结构化 commit subject

格式 axiomate:<messageId>:<label>，能反查到对话回合。

遵守 .gitignore

遍历工作目录时不把 node_modules/ 这种垃圾目录拍进快照。

2自动行为 无需手动

什么时候会自动拍快照

在每一次模型即将调用文件修改工具（Write / Edit / NotebookEdit）之前，Axiomate 会在影子仓库里写一个 commit。这一切默认开启，无需配置。

什么时候会自动跳过

条件行为
当前目录里文件数超过 50,000跳过这一次快照（成本太高）
单个文件大于 10 MB该文件不进 index，其它文件正常拍
当前目录是 / / ~ / 盘符根整个快照拒绝（防误伤）
跟上一次相比没有变化不写空 commit
系统里没装 git整个 checkpoint 子系统 no-op，不报错

什么时候会自动清理

Axiomate 会在你空闲 ≥1 分钟、且启动满 10 分钟之后， 在后台跑一次 prune。清理的内容包括：

  • 孤儿（orphan）：工作目录已经被你手动删掉的项目， 它们的 ref 会被删掉。
  • 过期（stale）：超过 14 天没碰过的项目 ref。
  • 超容量（size-cap）：所有快照加起来超过 500 MB 时，从最旧的开始裁。
  • 会话锚定（keep-ref）：如果某条即将被删的 ref 上 还有最近 28 天内 session 引用的 commit，会把它锚到 refs/axiomate/_keep/ 里，保住到对应 session 也过期为止。

清理至多 24 小时跑一次（用 .last_prune 标记控制），不会 重复执行。所有 prune 都是 fail-soft 的，遇到错误只记录、不抛出。

3用户命令 主动调用

/rewind — 回到某个对话回合

在 REPL 里输入 /rewind，弹出一个交互式选择器，列出本次 会话的每一回合。选中某一回合后：

  • 对话状态会回到那一回合（之后的消息消失）。
  • 被 axiomate 工具碰过的文件（Edit / Write / NotebookEdit，以及 axiomate 登记过的 Bash 写入）会从对应快照 checkout 回去。axiomate 从没碰过的文件原样保留。
注意：rewind 的作用范围是 tracked 路径 — 也就是本次会话里 axiomate 工具改过的文件。axiomate 从没碰过的文件 上，你的手工改动不会被动到。唯一会丢的情况：axiomate 改过文件 X、 你又手工改了 X，rewind 会把 X 回到快照那一刻的样子。如果某个文件 在目标回合时还不存在，rewind 会把它从磁盘上删掉。

/checkpoints — 状态、列表、清理

命令作用
/checkpoints
/checkpoints status
/checkpoints status N		 显示影子仓库总大小、每个项目的快照数和最近活动时间。可选位置参数 N 控制最多显示几行（默认 30，范围 1..500）。
/checkpoints list
/checkpoints list N		 列出当前项目的快照，按时间倒序。可选位置参数 N 控制最多显示几行（默认 30，范围 1..500）。
/checkpoints prune
/checkpoints prune force
/checkpoints prune keep-orphans		 立即跑一次清理。默认带 24h 节流；加 force 绕过。加 keep-orphans 跳过孤儿（工作目录消失的）那一遍，stale 和 size-cap 仍然跑。斜杠命令的参数是位置子词（不带 --）；拼错的词（如 frce）会被拒绝并报用法错误。命令行端保留 POSIX 风格 flag — 见下方。
/checkpoints clear 清空整个影子仓库（所有项目的所有快照）。不可恢复。

命令行版本

同样的子命令也可以从 axiomate 的命令行调用，不进 TUI： 

axiomate checkpoints status
axiomate checkpoints status --rows 50
axiomate checkpoints list
axiomate checkpoints list --rows 50
axiomate checkpoints prune --force
axiomate checkpoints prune --retention-days 30 --max-size-mb 1024
axiomate checkpoints prune --keep-orphans --force
axiomate checkpoints clear --force

每个子命令都拒绝多余的位置参数（输出 too many arguments）， 不会再静默吞掉。clear 在命令行端必须显式 --force， 因为它绕过了斜杠版本里的交互确认。

控制 status / list 显示几行

status 和 list 渲染的表格默认最多 30 行。想多看（或少看）几行，传一个 1..500 之间的整数。斜杠命令直接用位置参数 （/checkpoints status 50、/checkpoints list 50）； 命令行用 --rows N，跟其他 CLI flag 风格一致 （axiomate checkpoints status --rows 50）。 非整数或超界会被拒绝。

想让覆盖值长期生效，把 checkpointsStatusRows 写到 Settings 里（叫 Checkpoints rows shown，在 10/20/30/50/100/200/500 之间循环），或者 /config checkpointsStatusRows N，再或者直接编辑 ~/.axiomate.json。每次调用的位置参数（斜杠命令） 或 --rows N（CLI）永远优先于 config 里的值。

--keep-orphans — 临时保住"假死"工作目录的 ref

默认情况下，prune 会把工作目录已经从磁盘上消失的项目 ref 直接删掉。 但如果你只是临时拔了外置盘、改了路径名打算再改回来、或者计划晚点 重新 clone 到原路径，可以加 --keep-orphans（CLI）或 keep-orphans（斜杠命令）：这次 prune 跳过孤儿那一遍， stale 和 size-cap 照常跑。/checkpoints prune keep-orphans 在 REPL 里也支持。命中这个安全阀的项目数会以 Orphan refs skipped: N 的形式出现在输出里。

典型输出

Checkpoint base: C:\Users\you\.axiomate\checkpoints
Total size:      284.3 MB
  store/         284.3 MB
Projects:        7

  WORKDIR                                                        COMMITS    LAST TOUCH  STATE
  C:\public\workspace\axiomate                                       142          5m ago  live
  C:\public\workspace\hermes-agent                                    87          2h ago  live
  C:\Users\you\Desktop\old-prototype                                   3         18d ago  orphan

Note: 3 snapshots from 1 orphan workdir will be discarded on next prune.

Snapshot metrics (last 100):
  ok 96, no-changes 3, skipped 0, failed 1
  duration p50 142ms, p95 380ms

Snapshot metrics 块是最近最多 100 次 snapshot 尝试的滚动窗口 （last N 那个数字会随快照累计涨到 100）。窗口数据写在 ~/.axiomate/checkpoints/metrics.jsonl 里，重启不丢。 第一次 snapshot 之前这块整段不显示； p50 / p95 至少要 2 个 ok 样本才会 展示数字，否则显示 —。

5失败模式 兜底

Checkpoint 子系统的核心契约是fail-open — 任何一次 快照失败都不能阻塞主流程。这意味着：

  • 系统里没装 git → 整个子系统 no-op，不报错。
  • git 命令瞬时失败（EBUSY、超时） → 这一次跳过，下一回合自动重试。
  • 工作目录有 50000 个以上文件 → 跳过，避免 git add 卡死。
  • 磁盘满 / 权限不足 → 错误进 logForDebugging，对话继续。

跨工作目录的边界情况

如果你 /resume 了一个旧 session，但当前工作目录跟当时 不一样，/rewind 之前 axiomate 会用一行系统消息提示你：

  • reachable：快照在当前 ref 里，可以正常 rewind。
  • reachable-other-worktree：快照存在，但属于另一个 工作目录。提示你 cd 过去。
  • unreachable：快照已被 prune 删掉，rewind 会失败。

清理时的孤儿提醒

/checkpoints status 末尾会附一行类似： 

Note: 3 snapshots from 1 orphan workdir will be discarded on next prune.

如果你看到这行、并且那个工作目录的 commit 还有用（比如刚把外置硬盘 拔了），可以把工作目录复原到原路径，下一次 prune 会重新认它为 live。

6FAQ

会污染我的 git 仓库吗？

不会。所有 commit 都写到 ~/.axiomate/checkpoints/store/， 跟你项目自己的 .git/ 完全独立。git status、 git log、git push 全都看不到这些 commit。

占多少磁盘？

硬上限 500 MB（可配）。git 自己会做对象去重和 zlib 压缩，所以同一个 文件改 100 次，实际只多存了几次 delta。日常使用一般在 100-300 MB。

能跨机器同步吗？

不能。Checkpoints 是本机偷偷拍的快照，没设计跨机器的同步、加密、 签名机制。换机器就当回到没历史的状态。

能导出 / 导入吗？

暂时没有 checkpoints export / import。 影子仓库本身是一个标准的 bare git repo，懂 git 的用户可以直接 git --git-dir=~/.axiomate/checkpoints/store ... 自己玩。

怎么彻底关掉？

在 Settings 里把 fileCheckpointingEnabled 设成 false， 或者环境变量 AXIOMATE_CODE_DISABLE_FILE_CHECKPOINTING=1。 之前已经攒下的快照不会被自动清理，需要手动 /checkpoints clear。

/rewind 报错说 commit 找不到，怎么办？

多半是那个回合的 commit 已经被 prune 掉了（超过 14 天 + 当前 session 没引用）。这是预期行为。如果是关键节点，把对应工作目录的产物拷一份 出来另存，别只依赖 checkpoint。
