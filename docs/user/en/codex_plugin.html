Axiomate User Guide

Codex Plugin Installation and Usage

This guide explains how to install, refresh, and initialize the Codex plugin in Axiomate.

Purpose

The Codex plugin integrates the OpenAI Codex CLI into Axiomate for code review, background tasks, rescue investigations, and an optional pre-stop review gate.

The plugin marketplace name is openai-codex, the plugin name is codex, so the install target is written as codex@openai-codex.

Prerequisites

Before using the Codex plugin, Codex CLI must already be runnable on your machine. This plugin is not a built-in Codex implementation; it calls the locally installed Codex CLI and Codex app server.

  • A working node installation is required locally. If you want /codex:setup to install Codex automatically, a working npm is also required.
  • Codex CLI must be installed locally: npm install -g @openai/codex.
  • Codex CLI needs to be logged in or have an API key configured. You can run !codex login, or use !codex login --device-auth or !codex login --with-api-key.

/codex:setup checks these conditions. If Codex CLI is not installed and npm is available, it will ask whether to install now.

First-time Installation

First add the marketplace, then install the plugin and reload:

/plugin marketplace add axiomates/codex-plugin-cc
/plugin install codex@openai-codex
/reload-plugins
/codex:setup

/codex:setup checks your local Node.js, npm, Codex CLI, and Codex authentication status. If it reports you are not logged in, follow the prompt in the output to run !codex login.

Reinstall or Refresh

If you previously installed an older version, it is recommended to uninstall first and then reinstall:

/plugin uninstall codex
/plugin install codex@openai-codex
/reload-plugins
/codex:setup

If you just want to pull the latest version from the marketplace, you can also refresh the marketplace first and then update the plugin:

/plugin marketplace refresh
/plugin update codex
/reload-plugins
/codex:setup

Common Commands

  • /codex:setup: Check whether Codex CLI is available and show next steps.
  • /codex:status: View the current Codex runtime, review gate status, and background tasks.
  • /codex:review: Run a Codex code review on the current git workspace or branch.
  • /codex:adversarial-review: Run a review more focused on design critique and assumption challenging.
  • /codex:rescue: Delegate investigation, fix, or follow-up rescue tasks to Codex.
  • /codex:result: View results for completed Codex background tasks.
  • /codex:cancel: Cancel a running Codex background task.

Optional: Enable Pre-Stop Review Gate

If you want Codex to perform a review every time before stopping a session, you can enable the review gate:

/codex:setup --enable-review-gate

Disable the review gate:

/codex:setup --disable-review-gate

Note: The review gate runs a Codex review during the stop phase, which may increase wait time. It is recommended to confirm that Codex CLI and authentication are working properly before enabling it.
