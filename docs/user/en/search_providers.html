Configuring Your Search Engine in ~/.axiomate.json

Last updated: 2026-06-02. This guide explains how to configure Brave, Exa, Tavily, or SerpApi as search providers in the user-level ~/.axiomate.json, enabling Axiomate's WebSearch tool to perform web searches.

Quick Start

Axiomate reads search provider configuration from the user-level config file: ~/.axiomate.json. On Windows this is typically C:\Users\<your-username>\.axiomate.json. The search configuration lives under the top-level key searchProviders, alongside models, model, and other top-level fields.

A minimal configuration only requires a provider name, type, and apiKey. The example below configures an Exa search provider:

{
  "searchProviders": {
    "exa": {
      "type": "exa",
      "apiKey": "exa_..."
    }
  }
}
The config file must be strict JSON. Do not include comments, trailing commas, or single quotes. Some README examples use JSONC style—remove comments when copying into ~/.axiomate.json.

No restart is needed after saving. Axiomate watches the global config file for changes and will pick up new provider settings the next time WebSearch is invoked.

How It Works

  1. The current primary model must already be configured under models and model.routes. Without a primary model, WebSearch will not appear as an available tool.
  2. When WebSearch is called, Axiomate checks whether searchProviders exists and has at least one usable provider.
  3. Search results are normalized into titles and URLs, while preserving the snippet content returned by the provider for the model to reference.
  4. The model must cite search result sources as Markdown links in its final response.

The keys under searchProviders are names you choose—for example exa, brave, or company-search. The type field on each provider determines which implementation is used.

Supported Providers

type Service Default Endpoint Best For
brave-web-search Brave Search API https://api.search.brave.com/res/v1/web/search General web search, up to 20 results.
exa Exa Search API https://api.exa.ai/search Semantic search, news, papers, company and people lookups, up to 100 results.
tavily Tavily Search API https://api.tavily.com/search AI-oriented search with optional answer, raw content, news, or finance topics, up to 20 results.
serpapi SerpApi https://serpapi.com/search.json Aggregated results from Google and other engines, supports engine, region, language, and device parameters, up to 100 results.

Multiple Providers & Fallback

searchProviders supports multiple providers at once. Axiomate will try them in the order they appear in the JSON object, and the first one to return results successfully is used.

{
  "searchProviders": {
    "exa": {
      "type": "exa",
      "apiKey": "exa_..."
    },
    "tavily": {
      "type": "tavily",
      "apiKey": "tvly-..."
    },
    "serpapi": {
      "type": "serpapi",
      "apiKey": "serp_..."
    }
  }
}
Fallback rules: Errors such as network failures, authentication errors, rate limiting, service unavailability, or malformed responses will cause Axiomate to try the next provider. However, errors of type invalid_request do not trigger a fallback, since they typically indicate the search request itself is invalid and switching providers might mask the real issue.

If a provider's type is not supported, Axiomate skips the unusable configuration and continues to the next provider. When all providers are unavailable, WebSearch will report an aggregated error message.

Domain Filters

The WebSearch tool input supports allowed_domains and blocked_domains. These are parameters the model uses when calling the search tool, not provider config fields in ~/.axiomate.json.

Exa & Tavily

Native domain filtering support. Axiomate passes allowed and blocked domains into the provider request body and still performs a local filter on returned results as a safeguard.

Brave & SerpApi

Supported via an adapter. Axiomate translates allowed domains into site:example.com queries and blocked domains into -site:example.com, then filters the returned results locally.

Domains are normalized; for example https://docs.example.com/path matches on the hostname docs.example.com. example.com matches example.com and its subdomains.

Field Reference

Common Fields

Field Required Description
type Yes The search provider type. Currently supported: brave-web-search, exa, tavily, serpapi.
apiKey Yes The API key for the corresponding search service. The current search provider implementation reads this string directly; there is no apiKeyEnv field.
baseUrl No Overrides the default search endpoint. Only set this when using a proxy, corporate gateway, or compatible service.

Brave Fields

Field Values Description
country String Region code, e.g. US.
searchLang String Search language, sent as search_lang.
uiLang String UI language, sent as ui_lang.
count Number, 1 to 20 Number of results to return. Defaults to 10 when unset; values above 20 are clamped to 20.
safeSearch off, moderate, strict Safe search level, sent as safesearch.
extraSnippets Boolean When true, requests additional snippet content from Brave.

Exa Fields

Field Values Description
searchType auto, neural, fast, deep-lite, deep, deep-reasoning, instant Exa search type, sent as type in the request body.
category company, research paper, news, personal site, financial report, people Restricts results to a specific category.
userLocation String User location, e.g. US.
numResults Number, 1 to 100 Number of results to return. Defaults to 10 when unset; values above 100 are clamped to 100.
includeText Array of strings Requires result content to contain these text strings.
excludeText Array of strings Excludes results containing these text strings.
moderation Boolean Whether to enable the Exa moderation parameter.
highlightMaxCharacters Number, 1 to 4000 Maximum characters for highlight snippets. Defaults to 800 when unset; values above 4000 are clamped to 4000.

Tavily Fields

Field Values Description
searchDepth advanced, basic, fast, ultra-fast Search depth, sent as search_depth.
chunksPerSource Number Only sent when searchDepth is advanced; the current implementation hard-clamps this to 3.
maxResults Number, 1 to 20 Number of results to return. Defaults to 5 when unset; values above 20 are clamped to 20.
topic general, news, finance Search topic.
timeRange day, week, month, year, d, w, m, y Restricts the time range.
startDate / endDate String Start and end dates, sent as start_date and end_date.
includeAnswer Boolean, basic, advanced Whether to request a Tavily-generated answer. If returned, it is provided to the model as an additional snippet.
includeRawContent Boolean, markdown, text Whether to request raw content.
country String Country or region, e.g. united states.
autoParameters Boolean Whether to enable Tavily auto parameters.
exactMatch Boolean Whether to enable exact matching.
includeUsage Boolean Whether to request Tavily usage information in the response.
safeSearch Boolean Whether to enable safe search.

SerpApi Fields

Field Values Description
engine String Search engine. Defaults to google when unset.
googleDomain String Google domain, sent as google_domain.
hl String Language parameter.
gl String Country or region parameter.
location String Location parameter.
device String Device parameter, e.g. desktop.
safe active, off Safe search parameter.
num Number, 1 to 100 Number of results to return. Defaults to 10 when unset; values above 100 are clamped to 100.
noCache Boolean When true, requests SerpApi to bypass its cache.

Security Notes

  • ~/.axiomate.json is a user-level private configuration file that may contain both model API keys and search API keys.
  • Do not commit this file to a Git repository, and never paste its contents into public issues, chat logs, or logs.
  • If you need to share a configuration, share only the provider's type, baseUrl, and non-sensitive parameters, replacing apiKey with a placeholder.
  • If you suspect a key has been leaked, rotate or revoke the old key in the corresponding search service dashboard.

Troubleshooting

Symptom Common Cause Resolution
Shows No search providers are configured ~/.axiomate.json has no top-level searchProviders, or the object is empty. Add at least one provider and verify correct field names and JSON syntax.
Shows model not configured, cannot use WebSearch The current primary model is not in models, or the primary route has no valid primary. Configure models and model.routes first, following the model configuration docs.
Shows unsupported provider type The type value is misspelled, or a search service that is not yet implemented was specified. Change type to one of the four supported values: brave-web-search, exa, tavily, or serpapi.
Authentication failure or HTTP 401 / 403 The apiKey is incorrect, expired, lacks the relevant API permissions, or the provider plan does not include the endpoint. Verify the key, account quota, and endpoint permissions. Brave uses X-Subscription-Token, Exa uses x-api-key, Tavily uses a Bearer token, and SerpApi uses the api_key query parameter.
Rate limiting or quota errors Search service quota exhausted, concurrent usage too high, or plan limit reached. Reduce search frequency, or configure a second provider as a fallback.
Returns empty results Query is too narrow, domain filters are too restrictive, or region/category parameters are overly constrained. Remove category, timeRange, domain filters, and other restrictions, then retry.
Changes don't take effect after saving JSON syntax error, saved to the wrong path, or the current session has not yet picked up the new file. Confirm the file path is ~/.axiomate.json, check for trailing commas, and wait about 1 second before invoking search again.
Recommended approach: Start by configuring a single provider and verify that search works. Once it is stable, add a second provider for fallback. This makes it easier to tell which API key or parameter is causing the problem when troubleshooting.
