Goals — 跨回合的持久化目标

普通对话是一问一答 — 你说一句，AI 答一段，结束。Goal 把你的目标提升成跨回合的"长任务锚点"：你 /goal <描述> 设一个目标，AI 开始动手，每个回合做完后由一个轻量的"评审模型" 自动判断目标是否达成 — 没达成就喂一句续作 prompt 让它接着做，达成了 就停下来通知你。直到完成 / 你叫停 / 预算耗尽。中途你随时可以发消息抢占、 可以 /subgoal 加追加条件、可以 Ctrl+C 暂停， /resume 后目标状态完整保留。它解决的是 — AI 在长任务 里跑着跑着就忘了原目标。

1它怎么工作

Goal 是经典 "Ralph loop" 范式 — 把一个长目标拆成一组自动续作的小回合， 让 AI 在每个回合的成果上接着干。Axiomate 的实现是一个 1:1 端口自 hermes-agent 的 /goal 系统，所有"细节抗碎"的硬化（评审失败自动暂停 / Ctrl+C 暂停 / 空响应跳过 / 用户抢占等）都搬过来了。

每个回合发生的事

第 N 轮 AI 正常处理上一句 user message（可能是你设的 goal，也可能是上一轮的 续作 prompt），调工具、改文件、生成回复。
回合结束 Axiomate 抽出 AI 这一轮的最后一段文本，连同你设的目标和所有 subgoal， 发给评审模型（默认走你的 fastModel）问：{"done": <bool>, "reason": "..."}。
判 done 目标状态翻成 done，屏幕上显示 ✓ Goal achieved: <reason>，loop 结束。
判 continue turn 计数 +1。如果还没超预算（默认 20 轮），把一句续作 prompt [Continuing toward your standing goal] Goal: <原文> … 推回消息队列；REPL 下一个 idle tick 当它是一条普通的 user message 启动下一轮。
预算耗尽 turn 计数 ≥ 上限时自动 pause，提示 ⏸ Goal paused — 20/20 turns used. Use /goal resume to keep going, or /goal clear to stop.

关键设计

普通 user message

续作 prompt 走的是和你手敲完全一样的消息队列。不动 system prompt， 不动工具集 — prompt 缓存不破。

评审模型是辅助任务

评审跑在独立的 fastModel（推荐配 Haiku、Gemini Flash 这类便宜模型）， 不占主对话的 token 配额。

持久化在 JSONL

目标状态以 type:'goal-state' 条目写进会话日志， /resume 后自动恢复。和文件 checkpoints 一样是 append-only + last-wins 范式。

fail-open

评审模型出错（超时、API fail、解析失败）一律返回 continue， 不会因为评审挂掉就卡住对话；连续 N 轮解析失败才自动暂停（N 默认 10，可在 /config 改）。

用户优先

你中途敲一条新消息，它抢占续作 prompt — 这一轮处理你的 消息，评审在你的消息处理完之后再跑（恰好用户的消息也可能正好完成目标）。

Ctrl+C 干净退出

中断一个回合时直接 pause（不评审、不重排），状态记成 user-interrupted (Ctrl+C)。/goal resume 接着跑。

2UI 标识

Goal 在屏幕上出现的几个地方 — 这套 glyph 是这个系统的"识别码"， 值得记一下：

符号时机含义
⊙ /goal <text> 后 / /goal status 目标已设 / 正在进行中。Footer 也常驻 ⊙ Goal 3/20: write fib pill。
↻ 每个 continue 回合开始 / transcript 中的续作 prompt 循环继续。屏幕显示 ↻ Continuing toward goal (3/20): <reason>； transcript 里自动喂回的 user message 前会有一个 ↻ 让你认得出 "这不是我打的，是 goal loop 自动喂的"。
✓ 评审判 done / 你 /goal clear 目标达成或主动收尾。
⏸ 预算耗尽 / 评审挂了 N 轮（默认 10） / Ctrl+C / 你 /goal pause 暂停但保留状态。/goal resume 接着跑。Footer pill 变黄色。
▶ /goal resume 从暂停恢复到 active。预算默认会重置（保持 hermes 一致行为）。

Footer pill

只要 goal 处于 active 或 paused，输入框下方就常驻一条 pill： 

⊙ Goal 3/20: write a fibonacci function and verify with fib(10)
⏸ Goal paused: refactor auth — turn budget exhausted (20/20)

这个 pill 是 axiomate 在 hermes 之外多做的 — 让你在长 loop 里 不会"跑着跑着忘了自己设过什么"。

3用户命令

/goal

形式作用等价别名
/goal 或 /goal status 显示当前目标状态（active / paused / 无目标）+ turn 进度 + subgoal 数量。 —
/goal <描述> 设一个新目标，会替换之前的目标，并立刻把描述本身当作第一句 user message 启动第一轮（不需要再发送一次）。 —
/goal pause 显式暂停。保留状态，/goal status 还能看到，下次 turn 不会自动续作。 —
/goal resume 从暂停恢复到 active，默认重置 turn 预算。下一句任何 user input 都会触发评审。 —
/goal clear 清空目标。Footer pill 消失，loop 完全停止。 /goal stop、/goal done
小提示： /goal <描述> 之后不需要再敲一句 Enter。 命令本身会把描述当作第一句 user message 自动入队（这是和 hermes 一致的便利设计 — cli.py:9261 的 self._pending_input.put(state.goal)）。

4/subgoal — 中途追加准则

你设了 /goal 实现一个 fib 函数，跑了两轮发现 AI 漏了 unit test。 这时不需要 clear 重设 — 用 /subgoal 追加准则： 

/subgoal 跑完后必须有一个 fib.test.ts 覆盖边界 case
/subgoal 输出 fib(10) 的实际值到 console

下一轮评审会切换到 with-subgoals 模板： 

Goal:
<原 goal>

Additional criteria the user added mid-loop (all must also be
satisfied for the goal to be DONE):
- 1. 跑完后必须有一个 fib.test.ts 覆盖边界 case
- 2. 输出 fib(10) 的实际值到 console

Agent's most recent response:
...

Decision: For each numbered criterion above, find concrete evidence
in the agent's response that the criterion is satisfied. ...

评审被强制对每条 subgoal 单独找证据，不会因为 AI 喊一句"全部完成"就放行。

形式作用
/subgoal列出当前所有 subgoal（带 1-based 编号）。
/subgoal <文本>追加一条准则。
/subgoal remove <n>按 1-based 编号删一条。
/subgoal clear清空所有 subgoal（不影响主 goal）。

Subgoal 不会单独 kick 一轮 — 当前 turn 跑完后下次评审才生效。 没有"立刻让 AI 知道新准则"的语义，因为 Axiomate 不打断进行中的回合。 所以 subgoal 适合"补漏"，不适合"急停"。要急停用 /goal pause。

5评审模型配置

评审模型是 Goal 系统的"成本敏感点" — 它每轮都跑一次，你不会希望 它走 Opus 4.7 这种贵模型。Axiomate 的解析顺序是：

优先级来源说明
1 midModel axiomate 全局的"中等任务"模型 — memory selection、session search summary、 goal judge 都共用。这是 judge 的最佳归属。
2 fastModel 没配 midModel 时退到 fastModel（token 估算、session search 也用它）。
3 currentModel fallback — 用主对话模型。第一次 /goal set 会打一行黄色 warning 提醒你成本，之后不再提示（一次性）。

推荐配置（编辑 ~/.axiomate.json）

{
  "currentModel": "claude-opus-4-7",
  "fastModel": "claude-haiku-4-5-20251001",
  "midModel": "claude-sonnet-4-6",
  "goalsMaxTurns": 20,
  "goalsParseFailureLimit": 10
}

goalsParseFailureLimit 可以在 /config 里选（预设 3 / 5 / 10 / 20 / 50 / 0，0 = 完全禁用 parse-failure auto-pause， loop 只靠 turn budget 兜底）。其他几项 ~/.axiomate.json 手编。

没有"goal 专属"的模型字段。Judge 是 axiomate 的"side task" 之一，跟 memory selection、session search summary 共享 midModel/fastModel 这条链路 — 配一处受益多处。如果你只想配一个便宜模型， 建议直接配 fastModel 就够了。

⚠ Reasoning 模型注意： QwQ、DeepSeek-R1、Qwen3-thinking 这类会在 JSON 前输出 <think>…</think> 推理段的模型可以用， Axiomate 的解析器会自动剥掉 think 块再 parse。但它们 token 消耗大、慢， 没必要用它们做 goal-judge — 评审任务只需要"是/否 + 一句理由"，不需要推理。

评审 prompt 长什么样

给评审模型的固定 system prompt（端口自 hermes）： 

You are a strict judge evaluating whether an autonomous agent has
achieved a user's stated goal. ...

A goal is DONE only when:
- The response explicitly confirms the goal was completed, OR
- The response clearly shows the final deliverable was produced, OR
- The response explains the goal is unachievable / blocked / needs
  user input (treat this as DONE with reason describing the block).

Otherwise the goal is NOT done — CONTINUE.

Reply ONLY with a single JSON object on one line:
{"done": <true|false>, "reason": "<one-sentence rationale>"}

"blocked / needs user input" 也算 DONE — 这是一条保护：评审不会让 loop 在"AI 卡着等你回答"的状态下继续浪费 token。

6和其他系统的配合

/resume — 自动恢复

目标状态写在 JSONL 会话日志里。axiomate --continue 或 /resume <sessionId> 加载之前的会话时，目标会自动浮出来 — 进 session 就能看到 Footer pill 显示 ⊙ Goal 3/20: …。状态是 active 还是 paused 完全按你离开时的样子。

/rewind — 回退后 goal 还在

/rewind 是回退对话链或文件状态，goal 状态不参与 rewind chain（它是 pointer-style 元数据，不进 buildConversationChain）。 也就是说：回退到几轮前的对话位置，goal 还是当前的 goal — 它跟着 sessionId 走，不跟着 conversation chain 走。

/branch / --fork-session — fork 后 goal 不继承

fork 一个新 session 时，旧 session 的 goal-state 条目不会 被复制到新会话 — fork 过滤器只带 transcript message，新会话从"无目标"开始。 这是有意的：fork 通常是为了开一条平行实验线，背着旧 goal 跑不合理。

抢占 — 你的消息 vs 续作 prompt

Goal loop 跑到一半你打字发了一条新消息怎么办？两种情况：

  • 消息是 slash command（/subgoal add … 之类）： 不算"真实消息"，不抢占 goal — slash command 处理完后 goal 继续。
  • 消息是普通文本：抢占。Axiomate 的 stopHook 在评审前检查 消息队列，发现有非 slash 的真实消息就 defer — 你的消息先走，等它处理完 下一次 turn 末再重新评审。

评审 verdict 不污染对话上下文

↻ Continuing toward goal (3/20): reason 这类提示用 isMeta: true 标记，你看得见但不进下一轮的模型 prompt。这避免了"评审的 prose 把 prompt cache 打乱"的问题。 但续作 prompt 本身是真实 user message，会进对话历史 （否则 AI 根本不知道要继续）。

7失败模式

Ralph loop 的危险在于"评审模型烂掉了你完全感觉不到，token 一直烧"。 Axiomate 内置了几道刹车：

故障行为恢复
评审 API 超时 / 网络出错 当作 continue，不计入 parse failure。Loop 继续。 自动恢复，无需操作。
评审返回非 JSON / 空 reply 当作 continue，consecutiveParseFailures +1。 累计 N 次自动 pause（见下条，N 默认 10），过程透明。
评审连续 N 轮都返回乱码（N 默认 10，/config 可设 3/5/10/20/50/0） 状态翻 paused，提示 ⏸ Goal paused — the judge model (N turns) isn't returning the required JSON verdict. Set midModel or fastModel in ~/.axiomate.json to a stricter model. Then /goal resume to continue. 编辑 ~/.axiomate.json 换成 instruct 模型，/goal resume。
turn 预算耗尽（默认 20） 状态翻 paused，提示 ⏸ Goal paused — 20/20 turns used. /goal resume 重置预算继续，或 /goal clear 收尾。
用户 Ctrl+C 中断回合 立刻 pause（不评审、不续作）， 状态记 user-interrupted (Ctrl+C)。 /goal resume 接着跑。
AI 空响应（最后一轮一段 text 都没产生） 跳过评审、不计入 turn 预算（避免被 API 抖动消耗）。 自动恢复，无需操作。
评审模型自己也用主模型（fastModel/aux 都没配） /goal set 时会打 warning 提醒，loop 还是会跑， 只是花钱。 编辑 ~/.axiomate.json 加 fastModel。
真正的硬故障 — goal 跑飞。如果你设了一个语义模糊的目标 （比如 /goal 让这个项目变得更好），评审会永远说 continue，AI 会 一直改东西。turn 预算是兜底，但 20 轮 Opus 调用也不便宜。 建议 — goal 描述里写明"完成的判断标准"，比如 /goal 实现 fib 函数并跑通 fib.test.ts 的所有用例，而不是 /goal 写个 fib 函数。

8FAQ

Goal 默认开启吗？

Goal 不主动占用任何资源 — 你不敲 /goal，整个系统就静默。 不需要配置开关。

不设 fastModel 也能用吗？

能用，但每轮评审都会走主模型，/goal set 时 axiomate 会 打 warning 提醒你成本问题。强烈建议至少配一个 fastModel — 那是 session search、title 生成等多个功能共用的辅助通道，配了一次受益多处。

Goal 跑的时候我能干别的吗？

能。Axiomate 的消息队列设计就是"用户优先" — 你随时打字发消息， 下一轮处理你的消息，goal 评审在你的回合之后再跑。你可以一边让 goal 跑一边用 /checkpoints status、/rewind、/goal status 这类只读 slash command 检查状态。

20 轮预算太少 / 太多怎么改？

编辑 ~/.axiomate.json 设 goalsMaxTurns。这是全局默认， 也可以未来 per-goal override（hermes 那边支持，axiomate 暂未暴露 CLI 形式）。 数字调大要谨慎，配合 midModel 或 fastModel 一起设， 不然 Opus × 50 轮就是真烧钱。

能让 goal 自动跑而不显示在 transcript 里吗？

暂时不能。续作 prompt 是真实 user message，进 transcript、占 token、 /resume 会带回来 — 这是有意的：让你能完整审计 Ralph loop 跑了什么。 只有评审 verdict（↻ / ✓ / ⏸ 那些）是 isMeta: true， 不进 transcript chain。

能让 /goal 走自定义 prompt 模板吗？

目前不能。续作 prompt 和评审 prompt 都是 hermes 原版模板。如果你需要 domain-specific 评审（比如"代码必须通过 typecheck 才算 done"），更好的 做法是把这个判定写进 goal 描述，让评审模型按你的话评。

和 axiomate 的 /skills loop 是一回事吗？

不是。/loop 是定时跑一个 prompt（cron 风格），不评审； Goal 是事件驱动地跑直到某个语义条件满足。/loop 适合"每 5 分钟 babysit 一次 PR"，Goal 适合"做完一个完整的事再停"。

Goal 持久化怎么实现？跨 axiomate 进程重启还在吗？

在。Goal state 以 type:'goal-state' 条目 append-only 写进 会话 JSONL（~/.axiomate/projects/<hash>/<sessionId>.jsonl）， 跟你的对话历史在同一个文件里。重启 axiomate、--continue 会自动 把最新一条 goal-state 读出来恢复。

有没有"全局 goal 列表"能查看所有项目所有目标？

暂时没有。Goal 是 per-session 的，没有跨 session 索引。要查所有项目的 goal 状态需要自己 grep ~/.axiomate/projects/ 下所有 jsonl 的 goal-state 条目。这是个明确的"待加增强"。
