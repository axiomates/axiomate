配置自己的搜索引擎到 ~/.axiomate.json

更新时间：2026-06-02。本文说明如何把 Brave、Exa、Tavily 或 SerpApi 搜索服务配置到用户级 ~/.axiomate.json，让 Axiomate 的 WebSearch 工具可以联网搜索。

快速开始 支持的搜索引擎 多个 provider 与 fallback 域名过滤 字段参考 排错

快速开始

Axiomate 从用户级配置文件读取搜索引擎配置：~/.axiomate.json。Windows 上通常是 C:\Users\<你的用户名>\.axiomate.json。搜索配置是顶层字段 searchProviders，和 models、model 等字段平级。

最小配置只需要 provider 名、type 和 apiKey。下面示例配置一个 Exa 搜索引擎：

{
  "searchProviders": {
    "exa": {
      "type": "exa",
      "apiKey": "exa_..."
    }
  }
}
配置文件必须是严格 JSON。不要写注释、尾随逗号或单引号。README 中有些示例使用 JSONC 风格，复制到 ~/.axiomate.json 时需要去掉注释。

保存后通常不需要重启。Axiomate 会监听全局配置文件变更，下一次调用 WebSearch 时会读取新的 provider 配置。

它如何生效

  1. 当前主模型必须已经在 models 和 model.routes 中配置好。没有主模型时，WebSearch 不会作为可用工具出现。
  2. 调用 WebSearch 时，Axiomate 会检查 searchProviders 是否存在且至少有一个可用 provider。
  3. 搜索结果会被规范化成标题和 URL，同时保留 provider 返回的摘要片段供模型引用。
  4. 模型在最终回答中必须把搜索结果来源用 Markdown 链接列出来。

searchProviders 的键名由你自己决定，例如 exa、brave、company-search。真正决定使用哪个实现的是每个 provider 里的 type 字段。

支持的搜索引擎

type 服务 默认接口 适合场景
brave-web-search Brave Search API https://api.search.brave.com/res/v1/web/search 通用网页搜索，结果数量最多 20。
exa Exa Search API https://api.exa.ai/search 语义搜索、新闻、论文、公司和人物类检索，结果数量最多 100。
tavily Tavily Search API https://api.tavily.com/search 面向 AI 应用的搜索，可选 answer、raw content、新闻或金融主题，结果数量最多 20。
serpapi SerpApi https://serpapi.com/search.json Google 等搜索引擎结果聚合，支持 engine、地区、语言和设备参数，结果数量最多 100。

Brave 示例

{
  "searchProviders": {
    "brave": {
      "type": "brave-web-search",
      "apiKey": "BSA...",
      "baseUrl": "https://api.search.brave.com/res/v1/web/search",
      "country": "US",
      "searchLang": "en",
      "uiLang": "en-US",
      "count": 10,
      "safeSearch": "moderate",
      "extraSnippets": true
    }
  }
}

多个 provider 与 fallback

searchProviders 可以同时配置多个 provider。Axiomate 会按 JSON 对象里的书写顺序依次尝试，第一个成功返回结果的 provider 会被使用。

{
  "searchProviders": {
    "exa": {
      "type": "exa",
      "apiKey": "exa_..."
    },
    "tavily": {
      "type": "tavily",
      "apiKey": "tvly-..."
    },
    "serpapi": {
      "type": "serpapi",
      "apiKey": "serp_..."
    }
  }
}
fallback 规则：网络错误、鉴权错误、限流、服务不可用、响应格式异常等错误会继续尝试下一个 provider。invalid_request 类型错误不会 fallback，因为这通常说明搜索请求本身无效，换 provider 可能掩盖真正问题。

如果某个 provider 的 type 不受支持，Axiomate 会跳过这个不可用配置，继续尝试后面的 provider。所有 provider 都不可用时，WebSearch 会报告聚合后的错误信息。

域名过滤

WebSearch 工具输入支持 allowed_domains 和 blocked_domains。这是模型调用搜索工具时使用的参数，不是 ~/.axiomate.json 中的 provider 配置字段。

Exa 和 Tavily

原生支持域名过滤。Axiomate 会把允许域名和屏蔽域名传入 provider 请求体，并且仍会在本地再过滤一次返回结果。

Brave 和 SerpApi

通过适配器支持。Axiomate 会把允许域名转换成 site:example.com 查询，把屏蔽域名转换成 -site:example.com，然后本地过滤返回结果。

域名会被标准化，例如 https://docs.example.com/path 会按主机名 docs.example.com 匹配。example.com 会匹配 example.com 及其子域名。

字段参考

通用字段

字段 必填 说明
type 是 搜索 provider 类型。当前支持 brave-web-search、exa、tavily、serpapi。
apiKey 是 对应搜索服务的 API key。当前搜索 provider 实现直接读取这个字符串；没有 apiKeyEnv 字段。
baseUrl 否 覆盖默认搜索接口。只有在使用代理、企业网关或兼容服务时才需要设置。

Brave 字段

字段 取值 说明
country 字符串 地区代码，例如 US。
searchLang 字符串 搜索语言，会传为 search_lang。
uiLang 字符串 界面语言，会传为 ui_lang。
count 数字，1 到 20 返回结果数量。未设置时默认 10，超过 20 会被截断为 20。
safeSearch off、moderate、strict 安全搜索级别，会传为 safesearch。
extraSnippets 布尔值 设为 true 时请求 Brave 返回更多摘要片段。

Exa 字段

字段 取值 说明
searchType auto、neural、fast、deep-lite、deep、deep-reasoning、instant Exa 的搜索类型，会传为请求体里的 type。
category company、research paper、news、personal site、financial report、people 限制搜索类别。
userLocation 字符串 用户位置，例如 US。
numResults 数字，1 到 100 返回结果数量。未设置时默认 10，超过 100 会被截断为 100。
includeText 字符串数组 要求结果内容包含这些文本。
excludeText 字符串数组 排除包含这些文本的结果。
moderation 布尔值 是否启用 Exa moderation 参数。
highlightMaxCharacters 数字，1 到 4000 摘要高亮最大字符数。未设置时默认 800，超过 4000 会被截断为 4000。

Tavily 字段

字段 取值 说明
searchDepth advanced、basic、fast、ultra-fast 搜索深度，会传为 search_depth。
chunksPerSource 数字 仅在 searchDepth 为 advanced 时传入；当前实现会固定夹到 3。
maxResults 数字，1 到 20 返回结果数量。未设置时默认 5，超过 20 会被截断为 20。
topic general、news、finance 搜索主题。
timeRange day、week、month、year、d、w、m、y 限制时间范围。
startDate / endDate 字符串 起止日期，会传为 start_date 和 end_date。
includeAnswer 布尔值、basic、advanced 是否请求 Tavily 生成 answer。返回后会作为额外摘要提供给模型。
includeRawContent 布尔值、markdown、text 是否请求原始内容。
country 字符串 国家或地区，例如 united states。
autoParameters 布尔值 是否启用 Tavily 自动参数。
exactMatch 布尔值 是否启用精确匹配。
includeUsage 布尔值 是否要求 Tavily 返回 usage 信息。
safeSearch 布尔值 是否启用安全搜索。

SerpApi 字段

字段 取值 说明
engine 字符串 搜索引擎。未设置时默认 google。
googleDomain 字符串 Google 域名，会传为 google_domain。
hl 字符串 语言参数。
gl 字符串 国家或地区参数。
location 字符串 位置参数。
device 字符串 设备参数，例如 desktop。
safe active、off 安全搜索参数。
num 数字，1 到 100 返回结果数量。未设置时默认 10，超过 100 会被截断为 100。
noCache 布尔值 设为 true 时请求 SerpApi 不使用缓存。

安全提示

  • ~/.axiomate.json 是用户级私有配置，里面可能同时包含模型 API key 和搜索 API key。
  • 不要把这个文件提交到 Git 仓库，也不要贴到公开 issue、聊天记录或日志里。
  • 如果你需要共享配置，只共享 provider 的 type、baseUrl 和非敏感参数，把 apiKey 替换成占位符。
  • 如果怀疑 key 泄露，请到对应搜索服务后台轮换或撤销旧 key。

排错

现象 常见原因 处理方式
提示 No search providers are configured ~/.axiomate.json 没有顶层 searchProviders，或对象为空。 添加至少一个 provider，并确认字段和 JSON 语法正确。
提示模型未配置，无法使用 WebSearch 当前主模型不在 models 中，或主路由没有有效 primary。 先按模型配置文档配置 models 和 model.routes。
提示 unsupported provider type type 拼写错误，或使用了尚未实现的搜索服务。 把 type 改成四个受支持值之一：brave-web-search、exa、tavily、serpapi。
鉴权失败或 HTTP 401 / 403 apiKey 错误、过期、没有对应 API 权限，或 provider 后台套餐不支持该接口。 检查 key、账户额度和接口权限。Brave 使用 X-Subscription-Token，Exa 使用 x-api-key，Tavily 使用 Bearer token，SerpApi 使用 api_key 查询参数。
限流或 quota 错误 搜索服务额度耗尽、并发过高或套餐限制。 降低搜索频率，或配置第二个 provider 作为 fallback。
返回结果为空 查询太窄、域名过滤过严、地区或类别参数限制过强。 去掉 category、timeRange、域名过滤等限制后重试。
保存配置后仍未生效 JSON 语法错误、保存到错误路径，或当前会话尚未读取到新文件。 确认文件路径是 ~/.axiomate.json，检查没有尾随逗号；等待约 1 秒后再调用搜索。
推荐配置方式：先只配置一个 provider 并测试能否搜索；稳定后再添加第二个 provider 做 fallback。这样排错时更容易判断是哪一个 API key 或参数出了问题。
