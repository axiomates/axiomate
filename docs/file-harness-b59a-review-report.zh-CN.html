Executive Summary

结论先说

核心 FileRead / FileWrite / FileEdit 方向是成立的

当前主线设计是清楚的：Write 是全文件覆盖，因此已有文件必须 full Read；Edit 是精确替换，因此可以在 partial Read 后做小范围 edit，只要 mtime/content/sibling 写入没有 stale。

但还不能签字说整轮 harness 已完全无风险

HR1 的 resume/cold-start 重建问题已按“不改 JSONL、不读当前盘、解释历史工具语义”的方案修复；HR2 的 atomic fallback 范围也已收窄为“file tool strict，config/settings 受控 fallback”。未决项现在主要集中在 NotebookEdit 一致性、Bash simulated sed、registry alias、subagent killed/failed reminder。

建议保留：Write full-read guard 建议保留：Edit partial-read precise edit 已修复：Write/Edit resume reconstruction 建议保留：same-process sibling guard 需决策：NotebookEdit mtime fallback 需决策：Bash simulated sed guard
Decisions

已处理 2 个，剩余 5 个设计决策

已处理：Write canonicalization 与 resume/cold-start

Write 重建现在 canonicalize 历史输入；只有实际去掉 BOM 或归一 CR/CRLF 时才记录 runtime-only toolNormalization。Edit 重建只 replay 已知 full content，不读当前磁盘，也不改 JSONL。

已处理：atomic fallback 只给 config/settings 受控开启

writeFileSyncAndFlush_DEPRECATED 默认仍 strict；FileWrite/Edit 路径不 fallback。config/settings 显式 opt-in，且只在 temp 已写好后 rename 阶段出现 EPERM/EACCES/EBUSY 这类 lock 错误时 direct fallback。

1. NotebookEdit 是否补 content fallback

FileEdit/FileWrite 对 full-read mtime-only drift 有 content fallback；NotebookEdit 当前没有，可能误拒绝。

2. NotebookEdit stale 是 throw 还是返回 tool error

现在 stale execution 会 rethrow，更接近 FileEdit/FileWrite，但和旧 NotebookEdit 的 error payload 行为不同。

3. Bash simulated sed 是否属于 structured harness writer

它现在会 lock、preserve format、noteFileWrite，但不强制 read-before-write/stale guard。需要决定这是 permission-approved shell edit，还是严格 file harness edit。

4. registry 是否处理 realpath/case alias

当前 registry 用 path.normalize。symlink、Windows 大小写、路径 alias 可能绕过 sibling 检测。

5. killed/failed subagent 是否提醒父 context

完成路径会 append reminder，但 killed/failed subagent 可能已经改过文件。是否提醒需要确认。

Behavior Changes

行为变动列表

B01 · Existing-file Write 必须先 full Read建议保留

旧行为

已有文件要求读过，但普通 range Read 不可靠标记 partial，因此局部读后可能被当成足够覆盖写。

新行为

range Read 标记 isPartialView，已有文件 Write 缺 full Read 就拒绝；validation 和 execution 都检查。

B02 · Write 统一写 UTF-8 / LF / no BOM保留，resume 已修

旧行为

工具输出和 readFileState 主要使用模型传入的原始 content。

新行为

写盘、工具输出、LSP change、readFileState 都使用 canonical content。resume 重建也 canonicalize 历史 Write input；只有实际格式矫正时才加 runtime-only toolNormalization。

B03 · Edit 允许 partial Read 后精确编辑建议保留

旧行为

代码有 partial guard，但普通 range Read 没设置 flag，实际常常允许。

新行为

Read 会标 partial，但 Edit 不把 partial 当 unread；只在 mtime/content/sibling stale 时拒绝。

B04 · Edit 保留源文件格式建议保留

旧行为

BOM、lone CR、格式元数据处理不完整。

新行为

编辑时保留 encoding、BOM、majority line ending；混合换行平票默认 LF。

B05 · Read 记录 partial view 和 registry sequence建议保留

旧行为

普通 range read 没有强 partial 语义，也没有 process-local read/write sequence。

新行为

text/notebook read 会进入 registry；text range read 会标 isPartialView。

B06 · 重复 unchanged Read 有升级提示建议保留

旧行为

重复读同一未变范围只返回固定 FILE_UNCHANGED_STUB。

新行为

第二次添加 reread-loop 提示，第三次及以后添加 STOP；这是 model-facing 文本变化。

B07 · same-process sibling write 检测建议保留

旧行为

主要靠 mtime/content；如果 subagent 写后恢复 mtime，父 context 可能不知道。

新行为

process-local registry 记录 owner 和 sequence，能发现同进程其他 context 写入。

B08 · 同路径 structured write 串行化建议保留

旧行为

没有跨工具同路径 async mutex，只靠避免 stale check 和 write 之间 await。

新行为

FileWrite/FileEdit/NotebookEdit/simulated sed 都通过 path lock 串行。限制是不可重入。

B09 · atomic write fallback 收窄已修复

旧行为

任意 atomic 失败后清理 temp，然后 fallback 到直接写。

新行为

file tools 默认 strict；config/settings 显式 opt-in，且只在 rename 阶段 lock 类错误后 fallback。temp write、flush、chmod、非 lock rename 仍抛 atomic_write_failed。

B10 · failure taxonomy / metadata建议保留

旧行为

主要靠 message/errorCode，没有 typed reason。

新行为

增加 fileHarnessFailure，区分 not_read、stale_content、sibling_write_after_read 等。

B11 · FileEdit match failure escalation建议保留

旧行为

string_not_found / multiple_match 固定错误文本。

新行为

同一 read snapshot 下连续失败会提示 reread/STOP；telemetry 不含 path，只含 extension。

B12 · NotebookEdit 纳入 registry/lock部分需决策

旧行为

有 read-before-edit 和 mtime guard，但没有 sibling registry/path lock。

新行为

有 registry 和 lock，保留 BOM/line ending；但没有 content fallback，execution stale 改为 throw。

B13 · Bash simulated sed 参与 registry/lockguard 需决策

旧行为

格式检测分散，不记录 sibling write，不加 path lock。

新行为

保留格式、加锁、noteFileWrite；但仍不强制 read-before-write/stale guard。

B14 · arbitrary shell writes 仍不纳入 registry接受限制

旧行为

Bash/PowerShell 任意写文件不解析。

新行为

仍不解析。只有 internal simulated sed 参与。跨进程/任意 shell 写依赖 mtime/content/checkpoint。

B15 · subagent completion reminder失败路径需审

旧行为

subagent 改了父 context 已读文件，父 context 不一定被提醒。

新行为

完成路径 append reread reminder；但 killed/failed 路径是否提醒仍需审查。

B16 · cross-process registry 不做接受限制

旧行为

无跨进程 registry。

新行为

仍无跨进程 registry。tmux/separate CLI 依赖 mtime/content 和 checkpoint。

B17 · Read/notebook content normalization 扩大建议保留

旧行为

CRLF 处理较多，但 BOM/lone CR/notebook 归一化不完整。

新行为

strip leading BOM，CRLF/lone CR 归一到 LF；notebook Read 也归一化后 parse。

B18 · 防止把 FILE_UNCHANGED_STUB 写入文件建议保留

旧行为

模型可能把内部 read-dedup status 当成文件内容写回。

新行为

exact stub 和短 wrapper 会被拒绝；较大的真实文档引用 stub 仍允许。

B19 · checkpoint/fileHistory 测试稳定性调整测试-only

旧行为

部分 git-backed 测试 commit 数偏大，默认 timeout/cleanup 在 Windows 上容易抖。

新行为

设置 30s helper、减少无意义 commit 数、cleanup 加 retry。不要把它和 runtime 行为混在一起判断。
